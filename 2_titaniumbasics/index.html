Topic

Tools and SDKs

SDKs and Requirements

  • Native SDKs are required
  • Android
    • Oracle JDK
    • Android SDK (ADK)
    • Download platform & tool sets
    • Path and Environment settings
  • iOS
    • Xcode
    • A Mac! (Intel-based)
  • Future: BB10 & Windows Phone 8
Titanium's build scripts call out to the native tools to compile your app. Therefore, the native SDKs and tools are required. For Android, that means you'll need the Oracle JDK, the Android SDK, and you'll need to set some Path and Environment variables. You'll also need to download platform tools for each version of Android you want to develop for. For iOS, you'll need Xcode, which means you'll need a Mac. In most cases, you can use the latest version of the iOS SDK that comes with Xcode. You can download additional versions if needed.

Android SDK Files

  • Install by unzipping
  • Directories:
    • platforms
    • add-ons
    • platform-tools
    • tools

Android Platforms

  • android script
  • SDK v8 required
  • Must use Google APIs version
  • Current mix at:
    http://developer.android.com/

Path & Environment

  • OS X
    • Edit ~/.bash_profile (hidden, might not exist)
    • Add path to tools and platform-tools folders to your PATH
    • Restart terminal
  • Windows
    • Edit System Properties in Control Panel
    • Set PATH to both tools and platform-tools directories
    • Make sure you've also set the JDK's path and JAVAHOME variables
Setting paths like this helps you enter commands at the terminal window more easily. You'll configure Studio with the installation locations, so its functions will work even without these paths being set. On Windows, it's critical that you set the PATH and JAVAHOME variables properly so that the JDK will function.

iOS: Xcode

  • Not much setup, download and install
  • Location depends on OS X version
There's not much setup for Xcode. Unless you're running an older version of OS X, it will be installed into your personal Applications folder. On older systems, it was installed to the system's Applications folder.

Handy Tools

You'll probably want to access a couple of the Xcode tools directly. It can be convenient to be able to launch the iOS simulator and Instruments tools directly without opening Xcode. Use the steps shown here to access these tools, then create an Alias on your Dock or Desktop.

Studio Setup

  • Runs on OS X, Windows, and Linux
  • Dashboard will help you through SDK setup steps.
  • Configure SDK locations:
    • Open a tiapp.xml
    • Click Configure
    • Specify locations
  • Make sure to set a default Android SDK
    ... and choose a Google APIs version!
Studio nicely automates much of the setup for you. A couple of configurations you'll need to make include setting the path to your Xcode and Android SDKs and configuring the default Android SDK to use when building your project.

Creating & Configuring Projects

Creating a Project

Starting in Titanium Studio 3.0, you can select an Alloy or "Classic" template as the basis for a new project. Then, you must fill in the required details. We'll cover them on the next slide.

Tiapp.xml Editor - GUI mode

  • Set App ID, version, and more
  • Select Titanium SDK and platform support
  • Specify modules used
  • Cloud enable
Most of your configuration can be done in the GUI mode of the tiapp.xml editor.

Tiapp.xml Editor - XML mode

  • Configure orientation support
  • Platform-specific:
    • iOS: Info.plist keys, hardware requirements, etc.
    • Android: tooling, theme, custom versioning, etc.
  • See Guides: Building Native Applications > Appendices > Tiapp.xml and timodule.xml Reference
Some configurations must be done in XML view. We'll see later how to perform some of these tasks. But in general, settings like orientation support, hardware configurations, and tooling options are set in the XML view. See the referenced Guide for information about the various supported keys you can set in the tiapp.xml.

Topic

Building & Running Projects

Building a Project

  • Debug, Run, and Publish menus
  • Options based on targets configured in Tiapp.xml
  • Studio does a "preflight" check
  • For iOS installations, need to configure provisioning profiles
Simple build options are provided in Studio on the Debug, Run, and Publish menus. Which choices are available on those menus depends on the target platforms you've specified in the tiapp.xml file. Studio does a quick syntax check on your files and will prompt you if it detects errors. It won't catch logic or API errors, and it's not foolproof. But it saves you from wasting time when you've made a silly typo. We'll talk later about installing to iOS devices. Just know at this point you will need to set up your certificates and provisioning profiles to install to an iOS device.

Building at the Command Line

  • NodeJS-based command line interface
  • Integrate with custom code repositories, CI configurations, more
  • Integrate with editors other than Studio
The CLI provides full access to the Titanium tooling options. YOu can use it to build and run projects, manage your SDK versions, configure your Titanium environment, and more. It's great for integration with other systems and editors.

Build Options

  1. Build once
  2. Choose Run > Run Configurations
  3. Locate by platform & project
  4. Or use the CLI!
